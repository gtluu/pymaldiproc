pymaldiviz.dashboard.toggle_edit_preprocessing_parameters_modal(n_clicks_button, n_clicks_save, n_clicks_cancel, trim_spectrum_lower_mass_range, trim_spectrum_upper_mass_range, transform_intensity_method, smooth_baseline_method, smooth_baseline_window_length, smooth_baseline_polyorder, smooth_baseline_delta_mz, smooth_baseline_diff_thresh, remove_baseline_method, remove_baseline_min_half_window, remove_baseline_max_half_window, remove_baseline_decreasing, remove_baseline_smooth_half_window, remove_baseline_filter_order, remove_baseline_sigma, remove_baseline_increment, remove_baseline_max_hits, remove_baseline_window_tol, remove_baseline_lambda_, remove_baseline_porder, remove_baseline_repetition, remove_baseline_degree, remove_baseline_gradient, normalize_intensity_method, bin_spectrum_n_bins, bin_spectrum_lower_mass_range, bin_spectrum_upper_mass_range, peak_picking_method, peak_picking_snr, peak_picking_widths, peak_picking_deisotope, peak_picking_fragment_tolerance, peak_picking_fragment_unit_ppm, peak_picking_min_charge, peak_picking_max_charge, peak_picking_keep_only_deisotoped, peak_picking_min_isopeaks, peak_picking_max_isopeaks, peak_picking_make_single_charged, peak_picking_annotate_charge, peak_picking_annotate_iso_peak_count, peak_picking_use_decreasing_model, peak_picking_start_intensity_check, peak_picking_add_up_intensity, is_open)¶

Dash callback to toggle the preprocessing parameters modal window, populate the current preprocessing parameters saved in the global variable PREPROCESSING_PARAMS, and save any modified preprocessing parameters to PREPROCESSING_PARAMS if the Save button is clicked.

Parameters:

  • n_clicks_button – Input signal if the edit_preprocessing_parameters button is clicked.

  • n_clicks_save – Input signal if the edit_preprocessing_parameters_save button is clicked.

  • n_clicks_cancel – Input signal if the edit_preprocessing_parameters_cancel button is clicked.

  • trim_spectrum_lower_mass_range – Mass in daltons to use for the lower mass range during spectrum trimming.

  • trim_spectrum_upper_mass_range – Mass in Daltons to use for the upper mass range during spectrum trimming.

  • transform_intensity_method – Method to use for intensity transformation. Either square root (‘sqrt’), natural log (‘log’), log2 (‘log2’), or log10 (‘log10’) transformation.

  • smooth_baseline_method – Method to use for baseline smoothing. Either Savitzky Golay (‘SavitzkyGolay’), apodization (‘apodization’), rebin (‘rebin’), fast change (‘fast_change’), or median (‘median’).

  • smooth_baseline_window_length – The length of the filter window (i.e. number of coefficients).

  • smooth_baseline_polyorder – The order of the polynomial used to fit the samples. Must be less than window_length.

  • smooth_baseline_delta_mz – New m/z dimension bin width.

  • smooth_baseline_diff_thresh – Numeric change to remove.

  • remove_baseline_method – Method to use for baseline removal. Either statistics-sensitive non-linear iterative peak-clipping (‘SNIP’), TopHat (‘TopHat’), median (‘Median’), ZhangFit (‘ZhangFit’), modified polynomial fit (‘ModPoly’), or improved modified polynomial fit (‘IModPoly’).

  • remove_baseline_min_half_window – The minimum half window size used for morphological operations.

  • remove_baseline_max_half_window – The maximum number of iterations/maximum half window size used for morphological operations. Should be (w-1)/2 where w is the index-based width of feature or peak.

  • remove_baseline_decreasing – If False, will iterate through window sizes from 1 to max_half_window. If True, will reverse the order and iterate from max_half_window to 1 (gives smoother baseline).

  • remove_baseline_smooth_half_window – The half window to use for smoothing the data. If greater than 0, will perform a moving average smooth on the data for each window to give better results for noisy data.

  • remove_baseline_filter_order – If the measured data has a more complicated baseline consisting of other elements such as Compton edges, thena higher filter_order should be selected.

  • remove_baseline_sigma – The standard deviation of the smoothing Gaussian kernal. If None, uses (2 * smooth_half_window + 1) / 6.

  • remove_baseline_increment – The step size for iterating half windows.

  • remove_baseline_max_hits – The number of consecutive half windows that must produce the same morphological opening before accepting the half window as the optimum value.

  • remove_baseline_window_tol – The tolerance value for considering two morphological openings as equivalent.

  • remove_baseline_lambda – Affects smoothness of the resulting background. The larger the lambda, the smoother the background.

  • remove_baseline_porder – Adaptive iteratively reweighted penalized least squares for baseline fitting.

  • remove_baseline_repetition – How many iterations to run.

  • remove_baseline_degree – Polynomial degree.

  • remove_baseline_gradient – Gradient for polynomial loss. Measures incremental gain over each iteration. If gain in any iteration is less than this, further improvement will stop.

  • normalize_intensity_method – Method to use for normalizaton. Either total ion count (‘tic’), root mean squared (‘rms’), median absolute deviation (‘mad’), or square root (‘sqrt’).

  • bin_spectrum_n_bins – Number of bins to use.

  • bin_spectrum_lower_mass_range – Mass in daltons to use for the lower mass range during spectrum binning.

  • bin_spectrum_upper_mass_range – Mass in Daltons to use for the upper mass range during spectrum binning.

  • peak_picking_method – Method to use for peak picking. Either local maxima (‘locmax’) or continuous wavelet transformation (‘cwt’).

  • peak_picking_snr – Minimum signal-to-noise ratio required to consider peak.

  • peak_picking_widths – Required width of peaks in samples. If using ‘cwt’ method, used for calculating the CWT matrix. Range should cover the expected width of peaks of interest.

  • peak_picking_deisotope – Whether to perform deisotoping/ion deconvolution. Deisotoping performed using pyopenms.Deisotoper.

  • peak_picking_fragment_tolerance – The tolerance used to match isotopic peaks.

  • peak_picking_fragment_unit_ppm – Whether ppm or m/z is used as tolerance.

  • peak_picking_min_charge – The minimum charge considered.

  • peak_picking_max_charge – The maximum charge considered.

  • peak_picking_keep_only_deisotoped – If True, only monoisotopic peaks of fragments with isotopic pattern are retained.

  • peak_picking_min_isopeaks – The minimum number of isotopic peaks (at least 2) required for an isotopic cluster.

  • peak_picking_max_isopeaks – The maximum number of isotopic peaks (at least 2) required for an isotopic cluster.

  • peak_picking_make_single_charged – Whether to convert deisotoped monoisotopic peak to single charge.

  • peak_picking_annotate_charge – Whether to annotate the charge to the peaks in pyopenms.MSSpectrum.IntegerDataArray: ‘charge’.

  • peak_picking_annotate_iso_peak_count – Whether to annotate the number of isotopic peaks in a pattern for each monoisotopic peak in pyopenms.MSSpectrum.IntegerDataArray: ‘iso_peak_count’.

  • peak_picking_use_decreasing_model – Whether to use a simple averagine model that expects heavier isotopes to have less intensity. If False, no intensity checks are applied.

  • peak_picking_start_intensity_check – Number of the isotopic peak from which the decreasing model should be applied. <= 1 will force the monoisotopic peak to be most intense. 2 will allow the monoisotopic peak to be less intense than the 2nd peak. 3 will allow the monoisotopic peak and the 2nd peak to be less intense than the 3rd, etc. A number higher than max_isopeaks will effectively disable use_decreasing_model completely.

  • peak_picking_add_up_intensity – Whether to sum up the total intensity of each isotopic pattern into the intensity of the reported monoisotopic peak.

  • is_open – State signal to determine whether the edit_preprocessing_parameters_modal modal window is open.

Returns:

Output signal to determine whether the edit_preprocessing_parameters_modal modal window is open.
