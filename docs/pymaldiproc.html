peak_picking(method='cwt', widths=None, snr=3, deisotope=False, fragment_tolerance=0.05, fragment_unit_ppm=False, min_charge=1, max_charge=1, keep_only_deisotoped=False, min_isopeaks=2, max_isopeaks=10, make_single_charged=True, annotate_charge=False, annotate_iso_peak_count=False, use_decreasing_model=True, start_intensity_check=1, add_up_intensity=False)¶

Perform peak picking on mass spectrum. Documentation for various parameters is taken from their respective documentation pages.

Parameters:

  • method (str) – Method to use for peak picking. Either local maxima (‘locmax’) or continuous wavelet transformation (‘cwt’).

  • widths (float | None) – Required width of peaks in samples. If using ‘cwt’ method, used for calculating the CWT matrix. Range should cover the expected width of peaks of interest.

  • snr (int) – Minimum signal-to-noise ratio required to consider peak.

  • deisotope (bool) – Whether to perform deisotoping/ion deconvolution. Deisotoping performed using pyopenms.Deisotoper.

  • fragment_tolerance (float) – The tolerance used to match isotopic peaks.

  • fragment_unit_ppm (bool) – Whether ppm or m/z is used as tolerance.

  • min_charge (int) – The minimum charge considered.

  • max_charge (int) – The maximum charge considered.

  • keep_only_deisotoped (bool) – If True, only monoisotopic peaks of fragments with isotopic pattern are retained.

  • min_isopeaks (int) – The minimum number of isotopic peaks (at least 2) required for an isotopic cluster.

  • max_isopeaks (int) – The maximum number of isotopic peaks (at least 2) required for an isotopic cluster.

  • make_single_charged (bool) – Whether to convert deisotoped monoisotopic peak to single charge.

  • annotate_charge (bool) – Whether to annotate the charge to the peaks in pyopenms.MSSpectrum.IntegerDataArray: ‘charge’.

  • annotate_iso_peak_count (bool) – Whether to annotate the number of isotopic peaks in a pattern for each monoisotopic peak in pyopenms.MSSpectrum.IntegerDataArray: ‘iso_peak_count’.

  • use_decreasing_model (bool) – Whether to use a simple averagine model that expects heavier isotopes to have less intensity. If False, no intensity checks are applied.

  • start_intensity_check (int) – Number of the isotopic peak from which the decreasing model should be applied. <= 1 will force the monoisotopic peak to be most intense. 2 will allow the monoisotopic peak to be less intense than the 2nd peak. 3 will allow the monoisotopic peak and the 2nd peak to be less intense than the 3rd, etc. A number higher than max_isopeaks will effectively disable use_decreasing_model completely.

  • add_up_intensity (bool) – Whether to sum up the total intensity of each isotopic pattern into the intensity of the reported monoisotopic peak.

remove_baseline(method='SNIP', min_half_window=1, max_half_window=100, decreasing=True, smooth_half_window=None, filter_order=2, sigma=None, increment=1, max_hits=1, window_tol=1e-06, lambda_=100, porder=1, repetition=None, degree=2, gradient=0.001)¶

Perform baseline removal. Documentation for each algorithm is taken from their respective documentation pages.

Parameters:

  • method (str) – Method to use for baseline removal. Either statistics-sensitive non-linear iterative peak-clipping (‘SNIP’), TopHat (‘TopHat’), median (‘Median’), ZhangFit (‘ZhangFit’), modified polynomial fit (‘ModPoly’), or improved modified polynomial fit (‘IModPoly’).

  • min_half_window (int) – The minimum half window size used for morphological operations.

  • max_half_window (int) – The maximum number of iterations/maximum half window size used for morphological operations. Should be (w-1)/2 where w is the index-based width of feature or peak.

  • decreasing (bool) – If False, will iterate through window sizes from 1 to max_half_window. If True, will reverse the order and iterate from max_half_window to 1 (gives smoother baseline).

  • smooth_half_window (int | None) – The half window to use for smoothing the data. If greater than 0, will perform a moving average smooth on the data for each window to give better results for noisy data.

  • filter_order (int) – If the measured data has a more complicated baseline consisting of other elements such as Compton edges, thena higher filter_order should be selected.

  • sigma (float | None) – The standard deviation of the smoothing Gaussian kernal. If None, uses (2 * smooth_half_window + 1) / 6.

  • increment (int) – The step size for iterating half windows.

  • max_hits (int) – The number of consecutive half windows that must produce the same morphological opening before accepting the half window as the optimum value.

  • window_tol (float) – The tolerance value for considering two morphological openings as equivalent.

  • lambda (int) – Affects smoothness of the resulting background. The larger the lambda, the smoother the background.

  • porder (int) – Adaptive iteratively reweighted penalized least squares for baseline fitting.

  • repetition (int | None) – How many iterations to run.

  • degree (int) – Polynomial degree.

  • gradient (float) – Gradient for polynomial loss. Measures incremental gain over each iteration. If gain in any iteration is less than this, further improvement will stop.

  • window_length (int) – The length of the filter window (i.e. number of coefficients).

  • polyorder (int) – The order of the polynomial used to fit the samples. Must be less than window_length.

  • diff_thresh (float) – Numeric change to remove.

class pymaldiproc.classes.PMPTdfSpectrum(tdf_data, frame: int, mode: str, precursor=0, diapasef_window=None, profile_bins=0, encoding=64, exclude_mobility=False)¶

Bases: TdfSpectrum, PMPMethods

Class for parsing and storing spectrum metadata and data arrays from open format MALDI-TIMS-TOF and MALDI-TIMS-qTOF mass spectra in Bruker TDF files. Data parsing methods are inherited from pyTDFSDK.classes.TdfSpectrum. Preprocessing methods are inherited from pymaldiproc.classes.PMPMethods.

Parameters:

  • tdf_data (pyTDFSDK.classes.TdfData) – TdfData object containing metadata from analysis.tdf database.

  • frame (int) – ID of the frame of interest.

  • mode (str) – Data array mode, either “profile”, “centroid”, or “raw”.

  • precursor (int) – ID of the precursor of interest for ddaPASEF data. If specified, overrides the frame ID during data parsing.

  • diapasef_window (dict) – Dictionary containing a row of metadata from the pyTDFSDK.classesTdfData.analysis[‘DiaFrameMsMsWindows’] table required for parsing diaPASEF data.

  • profile_bins (int) – Number of bins to bin spectrum to.

  • encoding (int) – Encoding bit mode, either “64” or “32”
